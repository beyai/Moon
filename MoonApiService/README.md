# API
 - App启动时 使用公钥加密一个随机密钥，发送给服务器，服务器使用私钥解密后得到这个随机密钥并保存
 - App、服务端 后续数据都使用这个随机密钥对所有请求与响应的数据进行对称加密

# Websocket
- 房间每一次创建时随机生成一个密钥，当客户端连接上来时，使用客户商协商的解密加载数据下发房间密钥给客户端
- 连接成功后，服务器使用上一步的随机密钥，将为房间生成一个随机密钥加密下发给连接的客户端
- 客户端后续的所有消息都使用房间的这个密钥进行加密后 publish消息给所有连接到同一房间的客户端

# 服务端解密
- 解密过期：415 密钥已过期，需要重新生成新的密钥
- 解密失败：400 消息无法解密
- 刷新密钥：客户端重新随机生成密钥，并使用公钥加密，发送给服务器
